# Quick Task 260421-34c — Auto-scan + clickable IPs

## Goal

When the user navigates to the Geräte page (`/devices`), the network
scan for un-registered Shellys should kick off automatically, without
them having to click "Geraete suchen" first. Re-click stays available
for manual re-scan.

## Scope

One file: `src/components/devices/discovery-list.tsx`.

Add a `useEffect` that fires `handleScan` once on mount. Guard with a
`didAutoScanRef` so React StrictMode double-mount in dev doesn't double-
fire, and skip if a scan is already running.

No other behavior changes.

## Tasks

### T1 — Mount-effect auto-scan in DiscoveryList

- **File:** `src/components/devices/discovery-list.tsx`
- **Action:**
  - Import `useEffect` and `useRef`.
  - Add `const didAutoScanRef = useRef(false)` inside the component.
  - Add `useEffect(() => { if (!didAutoScanRef.current) { didAutoScanRef.current = true; void handleScan(); } }, [])` right after the state declarations / before `handleScan`.
  - Since `handleScan` is declared below the effect, define the effect via a `useEffect` that calls a captured `scanRef.current` OR just move the effect after `handleScan`. The simpler route: place the effect at the end of the function body before the `return`, after `handleScan` is declared (JS hoists `function` declarations, but `async function handleScan()` nested inside the component is NOT hoisted across the render body — need to place effect after the declaration, or wrap `handleScan` in a `useCallback` and reference it).
  - Cleanest pattern: convert `handleScan` into `useCallback` (stable ref), then `useEffect(() => { if (!didAutoScanRef.current) { didAutoScanRef.current = true; void handleScan(); } }, [handleScan])`.
- **Verify:** Browser — navigate to `/devices`, scan starts automatically. Repeat-navigation starts again (by design).
- **Done:** Page mount triggers scan once.
