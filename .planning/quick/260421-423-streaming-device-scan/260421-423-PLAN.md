# Quick Task 260421-423 ŌĆö Streaming device scan with progress bar

## Goal

A /24 subnet scan takes ~20 s today because it probes 254 IPs in batches
of 20 with a 1.5 s timeout. The user currently stares at "Scanne
Netzwerk..." for the whole time and only sees results at the very end.

Switch to incremental streaming:
- Discovered devices appear in the list as soon as each probe hits.
- A progress bar + `X / 254` counter shows how far along the scan is.

## Scope

- `src/modules/shelly/discovery-scanner.ts` ŌĆö add `onDevice` callback
  that fires the moment a probe returns a ScanResult.
- `src/app/api/devices/discover/stream/route.ts` (new) ŌĆö SSE endpoint
  that wraps `scanSubnet()`, forwarding `onProgress` + `onDevice` as
  SSE events and closing the stream on completion.
- `src/components/devices/discovery-list.tsx` ŌĆö switch from `fetch`
  to `EventSource`, render results incrementally, show a progress
  bar with scanned/total counts.

Old `/api/devices/discover` stays as a sync fallback in case any other
caller relies on it.

## Tasks

### T1 ŌĆö Scanner emits per-device events

- **File:** `src/modules/shelly/discovery-scanner.ts`
- **Action:** Extend `scanSubnet()` signature with
  `onDevice?: (device: ScanResult) => void`. Inside the batch loop,
  after `probeDevice` resolves, invoke `onDevice(result)` immediately
  for non-null results (before pushing into the `results` array).
  Keep `onProgress` exactly as-is.
- **Done:** Signature + per-device callback emit.

### T2 ŌĆö SSE endpoint

- **File:** `src/app/api/devices/discover/stream/route.ts` (new)
- **Action:**
  - `export const runtime = 'nodejs'` and `dynamic = 'force-dynamic'`.
  - GET handler returns a `ReadableStream` with headers
    `text/event-stream; Cache-Control: no-cache, no-transform;
    Connection: keep-alive; X-Accel-Buffering: no`.
  - Inside `start(controller)`:
    - `scanSubnet({ onDevice, onProgress })` where:
      - `onDevice` enqueues `event: device\ndata: <json>\n\n`.
      - `onProgress` enqueues `event: progress\ndata:
        {"scanned":N,"total":254}\n\n`.
    - After `scanSubnet` resolves: enqueue
      `event: done\ndata: {"total":N}\n\n` and close the controller.
  - Handle `request.signal.abort` by closing the controller
    (scanSubnet itself can't be aborted mid-flight; wrap in a cancel
    flag checked by callbacks so enqueue doesn't throw after close).
- **Done:** SSE stream runs end-to-end and emits device / progress /
  done events.

### T3 ŌĆö UI: EventSource + progress bar

- **File:** `src/components/devices/discovery-list.tsx`
- **Action:**
  - Add `progress` state `{scanned: number, total: number} | null`.
  - Rewrite `handleScan`:
    - Close any existing EventSource, reset `devices` + `relayState`
      + `progress`.
    - Open `new EventSource('/api/devices/discover/stream')`.
    - `addEventListener('device')`: parse JSON; `setDevices(prev =>
      [...prev, parsed])`; seed `relayState` with `parsed.output`.
    - `addEventListener('progress')`: `setProgress(parsed)`.
    - `addEventListener('done')`: `setScanning(false)`,
      `setHasScanned(true)`, close EventSource.
    - `onerror`: set error, close.
  - Render a progress bar (horizontal, blue fill, 2 px high) under
    the scan button while `scanning === true`, with `X / 254 IPs
    gepr├╝ft` below. Auto-hide after done.
  - List renders from state as events arrive ŌĆö no change to the row
    component beyond that. Keep auto-scan effect intact.
  - Cleanup: EventSource in a ref, close on unmount.
- **Done:** Rows appear one-by-one, progress bar fills, no button-
  mashing needed.
