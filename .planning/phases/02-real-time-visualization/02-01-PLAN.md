    **1. Create SSE endpoint** at `src/app/api/sse/power/route.ts` (per D-19 -- single global SSE stream):
    - Export `const dynamic = 'force-dynamic'` and `const runtime = 'nodejs'` (REQUIRED to prevent Next.js buffering -- see Pitfall 1 in RESEARCH.md)
    - Export async `GET(request: Request)` handler
    - Access `globalThis.__eventBus` (typed as EventBus)
    - Create a `ReadableStream` that:
      - In `start(controller)`: register handler on `eventBus.on('power:*', handler)` that encodes `data: ${JSON.stringify(reading)}\n\n` via TextEncoder and enqueues
      - Also register handler on `eventBus.on('online:*', handler)` that sends `event: online\ndata: ${JSON.stringify(event)}\n\n` (named event for online/offline)
      - On `request.signal.addEventListener('abort', ...)`: call `eventBus.off()` for both handlers, then `controller.close()` (wrapped in try/catch)
    - Return `new Response(stream, { headers })` with headers:
      - `Content-Type: text/event-stream`
      - `Cache-Control: no-cache, no-transform`
      - `Connection: keep-alive`
      - `X-Accel-Buffering: no` (prevents nginx proxy buffering)

    **2. Create client-side power stream hook** at `src/hooks/use-power-stream.ts`:
    - Add `'use client'` directive at top
    - Implement singleton EventSource pattern (module-level variables, NOT React state):
      - `let sharedEventSource: EventSource | null = null`
      - `let powerListeners = new Map<string, Set<PowerCallback>>()` (key = plugId or '*')
      - `let onlineListeners = new Set<OnlineCallback>()`
      - `let refCount = 0`
    - `getEventSource()`: creates EventSource to `/api/sse/power` if not exists or closed. Sets `onmessage` to parse JSON and distribute to matching plugId listeners AND wildcard `*` listeners. Sets named event listener for `'online'` events.
    - Export `usePowerStream(plugId: string | '*', onReading: PowerCallback)`:
      - In useEffect: increment refCount, call getEventSource(), add callback to powerListeners set for plugId
      - Cleanup: remove from set, decrement refCount, if 0 close EventSource and null it
      - Dependencies: `[plugId, onReading]`
    - Export `useOnlineStream(onEvent: OnlineCallback)`:
      - Same pattern but for onlineListeners set
    - Also maintain a module-level `latestReadings = new Map<string, PowerReading>()` that caches the most recent reading per plug (for sparkline data retention across navigation, per Open Question 1 in RESEARCH.md)
    - Export types: `type PowerCallback = (reading: PowerReading) => void`, `type OnlineCallback = (event: PlugOnlineEvent) => void`

    **3. Create sliding window hook** at `src/hooks/use-sliding-window.ts`:
    - Add `'use client'` directive
    - Define `WINDOW_POINTS` record (per D-15 time windows):
      - `'5m': 150` (5 * 60 / 2, at ~1 reading per 2 seconds from Shelly)
      - `'15m': 450`
      - `'30m': 900`
      - `'1h': 1800`
    - Export type `WindowKey = '5m' | '15m' | '30m' | '1h'`
    - Export `useSlidingWindow(windowKey: WindowKey = '15m')`:
      - Use `useRef<Array<[number, number]>>([])` for data (timestamp, watts pairs)
      - `maxPoints = WINDOW_POINTS[windowKey] ?? 450`
      - `push(timestamp, value)`: appends to ref, slices from end if exceeds maxPoints, returns current array
      - `getData()`: returns `dataRef.current`
      - `clear()`: resets to empty array
      - All callbacks wrapped in `useCallback` with `[maxPoints]` dependency

    **4. Update next.config.ts** -- add `transpilePackages: ['echarts', 'zrender']` to the config object (per Pitfall 5 in RESEARCH.md -- prevents SSR/hydration errors with ECharts). Keep existing `reactStrictMode: true`.
