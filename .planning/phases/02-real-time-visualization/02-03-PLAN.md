  <action>
    **1. Create historical readings API** at `src/app/api/devices/[id]/readings/route.ts`:
    - Export `const runtime = 'nodejs'`
    - Export async `GET(request: Request, context: { params: Promise<{ id: string }> })`:
      - Await `context.params` to get `id`
      - Parse URL search params for `window` (default '15m')
      - Calculate `since` timestamp: `Date.now() - windowMs` where windowMs is `{ '5m': 300_000, '15m': 900_000, '30m': 1_800_000, '1h': 3_600_000 }[window]`
      - Query: `db.select({ apower: powerReadings.apower, timestamp: powerReadings.timestamp }).from(powerReadings).where(and(eq(powerReadings.plugId, id), gte(powerReadings.timestamp, since))).orderBy(asc(powerReadings.timestamp)).all()`
      - Return `Response.json({ readings: rows.map(r => [r.timestamp, r.apower]) })` -- array of [timestamp, watts] tuples matching the chart data format
    - Import `db` from `@/db/client`, `powerReadings` from `@/db/schema`, `eq, and, gte, asc` from `drizzle-orm`

    **2. Create plug detail page** at `src/app/devices/[id]/page.tsx`:
    - This is a SERVER component (async function, loads plug data from DB)
    - Export default async function `PlugDetailPage({ params }: { params: Promise<{ id: string }> })`
    - Await params, look up plug from DB: `db.select().from(plugs).where(eq(plugs.id, id)).get()`
    - If not found, call `notFound()` from `next/navigation`
    - Load latest power reading for initial display: query most recent reading from `powerReadings` table
    - Layout:
      - Back link: `<Link href="/">` with left arrow + "Dashboard" text, styled as `text-neutral-400 hover:text-neutral-100 text-sm`
      - Header: plug name as `<h1 className="text-2xl font-bold text-neutral-100">`, online/offline dot indicator
      - Stats row: current watt value (from latest reading or '--'), relay status (on/off badge), total energy (from latest reading)
      - Chart section: `<PlugDetailChart plugId={id} />` -- client component wrapper
    - Styling: full-width chart container, dark theme consistent with dashboard (`bg-neutral-900 rounded-lg border border-neutral-800 p-6`)

    **3. Create client chart wrapper** at `src/app/devices/[id]/plug-detail-chart.tsx`:
    - Add `'use client'` directive
    - Props: `{ plugId: string }`
    - On mount: fetch historical data from `/api/devices/${plugId}/readings?window=${windowKey}`
    - Pass fetched data as `initialData` to `<PowerChart plugId={plugId} initialData={historicalData} />`
    - Re-fetch when window key changes
    - Handle loading state: show skeleton/placeholder while historical data loads
    - The PowerChart component handles live SSE data on top of the initial data

    **IMPORTANT:** The PowerChart component from Plan 02 needs to accept an `initialData` prop. The detail chart wrapper manages:
      1. Fetching `/api/devices/${plugId}/readings?window=15m`
      2. Calling `push(timestamp, watts)` for each historical point via sliding window
      3. SSE streaming continues appending new points
  </action>
