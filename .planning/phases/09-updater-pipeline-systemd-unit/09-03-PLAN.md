**From Plan 09-01 (live on dev server at localhost:3000):**
- `POST http://127.0.0.1:3000/api/internal/prepare-for-shutdown` returns 200 with `{"status":"drained","at":<ms>,"drainedPages":<n>,"pollersStopped":<n>}`
- After this call, HttpPollingService has no active pollers. Calling `pnpm dev` restart is needed to resume polling.

**From Phase 7 (live on dev server at localhost:3000):**
- `GET http://127.0.0.1:3000/api/version` returns `{"sha":"<hex|unknown>","shaShort":"<7>","buildTime":"<iso>","rollbackSha":null,"dbHealthy":true}`
- Note: in dev mode, `sha` may be `"unknown"` if `generate-version.mjs` hasn't populated it yet. That's OK for the dry run — we parse the field, we don't assert it matches anything.

**From Plan 09-02 (shell functions we will source):**
- `preflight_disk` reads `df -BM "${INSTALL_DIR}"` — we override `INSTALL_DIR` to a tmp dir
- `preflight_node`, `preflight_pnpm` read `node -v` / `pnpm -v` — safe to run as-is
- `preflight_git` runs `git status --porcelain` in the CWD — safe to run against the current repo
- `do_snapshot` writes to `${SNAPSHOT_DIR}/${CURRENT_SHA}.tar.gz` — we override to a tmp dir
- `health_probe <sha>` loops curl against `${APP_URL}/api/version` — we override APP_URL to localhost:3000

**Key sourcing trick:** `run-update.sh` is self-contained and calls `main "$@"` at the bottom. We want the helpers WITHOUT running main. Strategy: source it with a guard. We set an env var `RUN_UPDATE_DRY_RUN=1` before sourcing; add a guarded return at the bottom of run-update.sh... ❌ NO — that modifies Plan 09-02's locked file. Instead we use a different approach: use `bash -c` to extract the helper function definitions via `declare -f` after sourcing in a subshell with `main() { :; }` defined BEFORE sourcing (which silently overrides the real main), OR we simply read the helper function bodies via `sed` and eval them in the harness.

The cleanest solution: source `run-update.sh` in a mode where `main` is pre-stubbed. Because bash function definitions are evaluated top-to-bottom, and `main "$@"` is the LAST line, we can ALSO short-circuit by defining a local `main` function in the harness that does nothing BEFORE we source run-update.sh. But bash would still call the real main (sourced definition wins). The only reliable way is to source a modified copy of run-update.sh OR to strip the last line with sed at source time.

**Chosen strategy:** the harness uses `sed` to strip the final `main "$@"` line from run-update.sh and sources the result via process substitution. This keeps Plan 09-02's file untouched.

```bash
source <(sed '/^main "\$@"$/d' scripts/update/run-update.sh)
```

This sources everything EXCEPT the `main "$@"` trigger line, giving us access to every helper function without running the pipeline. The `trap ... ERR` / `trap on_exit EXIT` lines WILL be installed — the harness disables them right after sourcing with `trap - ERR; trap - EXIT`.

Also: the sourced script runs `mkdir -p "${STATE_DIR}"` and `exec 9>"${LOCK_FILE}"` + `flock -n 9` at the top. Those fail on dev because `/opt/charging-master/.update-state/` doesn't exist and we're not root. The harness MUST override `INSTALL_DIR`/`STATE_DIR`/`LOCK_FILE` BEFORE sourcing. But bash `readonly` on the sourced script prevents that.

**Revised strategy:** the harness sources a filtered version of run-update.sh that ALSO strips the `readonly` keyword and the `flock` block. The harness's sed filter:

```bash
source <(
  sed -e '/^main "\$@"$/d' \
      -e 's/^readonly /# DRY-RUN:stripped readonly: /' \
      -e '/^exec 9>/,/^fi$/d' \
      scripts/update/run-update.sh
)
```

This strips:
1. The `main "$@"` invocation
2. The `readonly` keyword (so the harness can reassign constants)
3. The entire `exec 9>...` through `fi` block (the flock guard)

The harness then resets the constants to tmp-dir values BEFORE calling any helpers, and manages traps explicitly.

**Alternative considered (rejected):** copy run-update.sh to a tmp file, patch it, source the tmp file. This is simpler but leaves artifacts on disk and complicates cleanup.

**Final fallback (if sed filtering turns out fragile):** if the executor finds the sed approach too brittle in practice (e.g., because a new readonly line is added and the harness breaks), the harness can fall back to a self-contained inline implementation of the helper functions, explicitly comment-linked to run-update.sh. The goal is a working dry run, not stylistic purity.

<pitfall_mitigation>
- **P18 (silent success):** The dry run is itself a mitigation — we catch cases where the shell script parses but fails at runtime due to a typo in a variable name, a missing sed filter, or a subtle bash quoting bug. A human walks through the output and confirms each expected behavior, rather than relying on a green CI check that might not exercise the right code path.
- **Cross-platform risk:** macOS `df` and `du` have different flags from GNU `df`/`du`. `df -BM` is a GNU extension. If the dev machine is macOS, `preflight_disk` will fail with `df: illegal option -- B`. The harness detects this and logs a "macOS limitation: preflight_disk cannot be dry-run on macOS, structural review only" message rather than failing outright. This is explicitly acceptable because the LXC is Debian 13 and `df -BM` works there.
</pitfall_mitigation>

```bash
#!/usr/bin/env bash
# =============================================================================
# Phase 9 dry-run harness (DEV ONLY — DO NOT SHIP TO LXC)
# =============================================================================
# Exercises the parts of scripts/update/run-update.sh that can safely run on
# a developer's machine against a running `pnpm dev` server on port 3000.
#
# Usage:
#   1. Start the dev server in another terminal: `pnpm dev`
#   2. Run this script from the project root: `./scripts/update/dry-run-helpers.sh`
#
# What it exercises:
#   - preflight_{disk,node,pnpm,git}
#   - do_snapshot (against /tmp/cm-dry-run)
#   - do_drain (POST to localhost:3000)
#   - health_probe (GET from localhost:3000)
#
# What it does NOT exercise:
#   - do_stop / do_start (no systemctl on dev)
#   - do_fetch / do_reset / do_install / do_clean_build / do_build
#     (would mutate the working tree)
#   - do_rollback_stage1 / do_rollback_stage2
#   - db_* helpers (would write to the dev DB)
#   - state_set_* helpers (would mutate dev state.json)
#   - pushover_send (would send a real notification)
# =============================================================================

set -uo pipefail  # NOTE: NO -e — we want to keep running past individual failures

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
REPO_ROOT="$(cd "${SCRIPT_DIR}/../.." && pwd)"
RUN_UPDATE="${SCRIPT_DIR}/run-update.sh"
SCRATCH="/tmp/cm-dry-run"
DEV_URL="http://127.0.0.1:3000"

GREEN='\033[0;32m'
YELLOW='\033[1;33m'
RED='\033[0;31m'
NC='\033[0m'

pass() { echo -e "${GREEN}[PASS]${NC} $*"; }
fail() { echo -e "${RED}[FAIL]${NC} $*"; }
warn() { echo -e "${YELLOW}[WARN]${NC} $*"; }
info() { echo "[INFO] $*"; }

if [ ! -f "${RUN_UPDATE}" ]; then
    fail "run-update.sh not found at ${RUN_UPDATE}"
    exit 1
fi

info "Preparing scratch directory ${SCRATCH}"
rm -rf "${SCRATCH}"
mkdir -p "${SCRATCH}/.update-state/snapshots"
mkdir -p "${SCRATCH}/data"
# Fabricate a tiny fake db file so the snapshot tarball has something to include
echo "dry-run-stub" > "${SCRATCH}/data/charging-master.db"

info "Sourcing run-update.sh helpers (filtered)"
# Strip:
#   1. The `main "$@"` trigger line so sourcing does NOT run the pipeline
#   2. The `readonly` keyword so we can reassign constants
#   3. The flock block (exec 9>... flock ... fi) because /opt/charging-master
#      does not exist on dev and we are not root
# shellcheck disable=SC1090
source <(
    sed -e '/^main "\$@"$/d' \
        -e 's/^readonly /# DRY-RUN:stripped-readonly: /' \
        -e '/^exec 9>/,/^fi$/d' \
        "${RUN_UPDATE}"
)

# Disable the ERR/EXIT traps that the sourced script installed — we want to
# handle failures inline so the harness can continue past a single failed helper.
trap - ERR
trap - EXIT

# Override the constants to point at the scratch directory + dev server.
INSTALL_DIR="${SCRATCH}"
STATE_DIR="${SCRATCH}/.update-state"
SNAPSHOT_DIR="${STATE_DIR}/snapshots"
DB="${SCRATCH}/data/charging-master.db"
APP_URL="${DEV_URL}"
CURRENT_SHA="dryrun$(date +%s)"
ROLLBACK_SHA="${CURRENT_SHA}"
CURRENT_STAGE="dry-run"
RUN_ID=""

# Override helpers that would touch the real DB or state.json with no-ops.
db_start_run()    { info "db_start_run: skipped (dry run)"; RUN_ID=1; }
db_update_stage() { info "db_update_stage[$1]: skipped (dry run)"; }
db_finish_run()   { info "db_finish_run[$1]: skipped (dry run)"; }
state_set_installing() { info "state_set_installing: skipped (dry run)"; }
state_set_success()    { info "state_set_success: skipped (dry run)"; }
state_set_rolled_back() { info "state_set_rolled_back: skipped (dry run)"; }
pushover_send() { info "pushover_send[$1]: skipped (dry run)"; }

# ---------------------------------------------------------------------------
# Test 1: preflight helpers
# ---------------------------------------------------------------------------
info "=== Test 1: preflight helpers ==="

# preflight_disk uses `df -BM` which is GNU-only. On macOS it fails — that is
# expected and not a real problem.
if df -BM "${INSTALL_DIR}" >/dev/null 2>&1; then
    if preflight_disk 2>&1; then
        pass "preflight_disk returned success"
    else
        fail "preflight_disk returned failure on a tmp dir with plenty of space"
    fi
else
    warn "preflight_disk: df -BM not supported (macOS?) — skipping, verify structurally"
fi

if preflight_node 2>&1; then
    pass "preflight_node returned success"
else
    fail "preflight_node returned failure — is Node 22+ installed?"
fi

if preflight_pnpm 2>&1; then
    pass "preflight_pnpm returned success"
else
    fail "preflight_pnpm returned failure — is pnpm 10+ installed?"
fi

# preflight_git runs in the CWD which is the real project root. We expect a
# clean or near-clean tree; if it fails, print the reason but don't abort.
(cd "${REPO_ROOT}" && preflight_git 2>&1) && pass "preflight_git: clean tree" || warn "preflight_git: working tree has unexpected changes (not fatal for dry run)"

# ---------------------------------------------------------------------------
# Test 2: snapshot tarball creation + retention
# ---------------------------------------------------------------------------
info "=== Test 2: do_snapshot ==="

# Create a few fake old snapshots so retention pruning has something to do
touch "${SNAPSHOT_DIR}/old1.tar.gz"
touch "${SNAPSHOT_DIR}/old2.tar.gz"
touch "${SNAPSHOT_DIR}/old3.tar.gz"
touch "${SNAPSHOT_DIR}/old4.tar.gz"
sleep 1  # ensure the new one has a later mtime

# do_snapshot runs `tar -C "${INSTALL_DIR}" .` which requires some content
# beyond just data/. Put something harmless in there.
echo "dry run" > "${INSTALL_DIR}/README.txt"

if do_snapshot 2>&1; then
    if [ -f "${SNAPSHOT_DIR}/${CURRENT_SHA}.tar.gz" ]; then
        pass "do_snapshot: tarball created at ${SNAPSHOT_DIR}/${CURRENT_SHA}.tar.gz"
        info "  size: $(du -h "${SNAPSHOT_DIR}/${CURRENT_SHA}.tar.gz" | cut -f1)"
    else
        fail "do_snapshot: exited 0 but tarball is missing"
    fi

    remaining=$(find "${SNAPSHOT_DIR}" -name '*.tar.gz' | wc -l | tr -d ' ')
    info "  snapshots in dir after retention prune: ${remaining}"
    if [ "${remaining}" -le 3 ]; then
        pass "do_snapshot: retention kept ${remaining} ≤ 3 snapshots"
    else
        warn "do_snapshot: retention kept ${remaining} snapshots — expected ≤3 (may be OK if initial count was different)"
    fi
else
    fail "do_snapshot: exited non-zero"
fi

# ---------------------------------------------------------------------------
# Test 3: drain endpoint (requires `pnpm dev` running)
# ---------------------------------------------------------------------------
info "=== Test 3: do_drain (requires dev server on ${DEV_URL}) ==="

if curl -sf --max-time 2 "${DEV_URL}/api/version" >/dev/null 2>&1; then
    info "dev server is reachable"
    if do_drain 2>&1; then
        pass "do_drain: endpoint returned 200"
    else
        fail "do_drain: endpoint did not return 200 — check the dev server logs"
    fi
else
    warn "dev server NOT reachable at ${DEV_URL} — skipping drain test"
    warn "start the dev server with 'pnpm dev' and re-run this harness"
fi

# ---------------------------------------------------------------------------
# Test 4: health_probe (requires `pnpm dev` running)
# ---------------------------------------------------------------------------
info "=== Test 4: health_probe ==="

if curl -sf --max-time 2 "${DEV_URL}/api/version" >/dev/null 2>&1; then
    # Grab the current sha from the live endpoint and use that as the target.
    # This means health_probe should return 0 immediately because sha matches.
    live_body=$(curl -sf --max-time 2 "${DEV_URL}/api/version")
    live_sha=$(echo "${live_body}" | grep -o '"sha":"[a-f0-9]*"' | cut -d'"' -f4)
    info "live sha: ${live_sha:-<unknown>}"

    if [ -n "${live_sha}" ] && [ "${live_sha}" != "unknown" ]; then
        if health_probe "${live_sha}" 2>&1; then
            pass "health_probe: returned 0 for matching live sha"
        else
            fail "health_probe: did not match on live sha ${live_sha}"
        fi
    else
        warn "live sha is empty or 'unknown' — skipping health_probe positive test"
    fi

    # Negative test: probe for a sha that cannot possibly match. Should return 1
    # after the 60s deadline — we trim to 5s via a timeout wrapper so the dry
    # run does not hang.
    info "health_probe negative test (expect timeout after ~5s via timeout wrapper)"
    if timeout 5 bash -c 'health_probe 0000000000000000000000000000000000000000' 2>&1; then
        fail "health_probe: returned 0 for an impossible sha (should have timed out)"
    else
        pass "health_probe: correctly did not match for an impossible sha (exit via timeout)"
    fi
else
    warn "dev server not reachable — skipping health_probe test"
fi

# ---------------------------------------------------------------------------
# Summary
# ---------------------------------------------------------------------------
echo ""
info "=== Dry run complete ==="
info "Scratch dir: ${SCRATCH} — remove with: rm -rf ${SCRATCH}"
info "NOTE: the drain test stopped your dev server's HttpPollingService."
info "      Restart 'pnpm dev' to resume normal dev operation."
```
