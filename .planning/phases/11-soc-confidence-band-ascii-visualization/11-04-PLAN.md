<objective>
Surface the SOC band in the UI: a new `SocBandIndicator` React client component reads ChargeStateEvent.socMin / socMax / socBandConfidence / socAsciiBar from the live SSE stream, drives `--soc-min` and `--soc-max` CSS variables on a parent element so the browser composites a smooth band-narrowing animation (per RESEARCH.md "CSS wins over ECharts"), and falls back to an inline `<pre>{socAsciiBar}</pre>` when no JS or when band fields are missing. The dashboard charge banner integrates the indicator. A new `ChargingSettings` component drops into the Settings page using the existing `useAutoSave` autosave pattern; a radio toggle controls `charging.stopMode` (default `aggressive`), and a collapsed/advanced number input controls `charging.bandThreshold` (default value pinned by Plan 11-01's empirical calibration).

Purpose: SOCB-06 calls for live band visualization, no-JS fallback, and configurability. This plan closes the user-facing loop — the band that propagates server-side, narrows on matcher runs, and feeds Pushover via Plan 11-03 now also appears on the user's dashboard tab in real time, and the user can toggle the stop policy without an editor.

Output: A new `SocBandIndicator` component + RTL test; a charge-banner update wiring it in; a new `ChargingSettings` component on the Settings page + a REQUIRED component test (W6 closed — no "or skip" alternative); SSE route verified to passthrough the new fields (it already JSON-stringifies the whole event opaquely — the active-session-replay synthesis path needs band hydration too).
</objective>
