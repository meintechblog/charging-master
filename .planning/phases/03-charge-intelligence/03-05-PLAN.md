    **1. ChargeBanner (src/components/charging/charge-banner.tsx):**
    'use client' component. Props: `plugId: string`.

    Uses `useChargeStream(plugId, ...)` to receive real-time charge state events. Maintains local state for current session info (state, profileName, confidence, estimatedSoc, targetSoc, sessionId).

    Per D-33: When state is 'matched', 'charging', or 'countdown', show banner:
    - Layout: Full-width card at top of plug area, bg-neutral-800 border-l-4 border-blue-500
    - Content: "[profileName] erkannt ([confidence]% Confidence) -- Ladevorgang gestartet, Ziel: [targetSoc]%"
    - SOC progress bar: Horizontal bar showing estimatedSoc out of targetSoc, blue-500 fill
    - Current SOC display: Large text "[estimatedSoc]%"

    Per D-34: Override controls (always visible in banner):
    - "Profil aendern" dropdown: Fetch profiles from /api/profiles, select to override. On select, PUT /api/charging/sessions/[sessionId] with new profileId.
    - "Ziel-SOC" inline: SocButtons component, on change PUT /api/charging/sessions/[sessionId] with new targetSoc.
    - "Abbrechen" button: POST /api/charging/sessions/[sessionId]/abort. Red text, confirm on click.

    Per D-39: No separate emergency stop button -- existing relay toggle handles that.

    When state is 'detecting': Show subtle banner "Gerät wird erkannt..." with spinner.
    When state is 'idle' or no session: Don't render anything.
    When state is 'stopping' or 'complete': Show success message "Ladevorgang abgeschlossen bei [estimatedSoc]%" with green accent, auto-dismiss after 10 seconds.

    **2. CountdownDisplay (src/components/charging/countdown-display.tsx):**
    Per D-37: Shown when charge state is 'countdown' (last 5% before target).
    Props: `estimatedSoc: number, targetSoc: number`.
    Display: Large centered text showing remaining percentage: "[targetSoc - estimatedSoc]% verbleibend".
    Visual: Animated ring or progress circle showing progress from (target-5) to target.
    Color: Transitions from blue-500 to green-500 as it approaches target.
    Integrate into ChargeBanner (replaces the normal SOC display when in countdown state).

    **3. UnknownDeviceDialog (src/components/charging/unknown-device-dialog.tsx):**
    Per D-36: Shown when ChargeMonitor emits state='detecting' and no match is found (or when explicit 'unknown' event).
    Props: `plugId: string, sessionId: number, onClose: () => void`.
    Two action buttons:
    - "Jetzt anlernen": Navigate to /profiles/learn?plugId=[plugId] (starts learn wizard with plug pre-selected)
    - "Bestehendes Profil zuweisen": Show dropdown of existing profiles (fetch /api/profiles). On select, PUT /api/charging/sessions/[sessionId] with selected profileId.
    Modal overlay with dark backdrop, centered card, bg-neutral-800.
  </action>

    **2. Plug detail page (src/app/plugs/[id]/page.tsx):**
    Import ChargeBanner.
    Render `<ChargeBanner plugId={id} />` at the top of the page content.

    Per VIZL-03: When the plug has an active charge session with a matched profile:
    - Fetch the reference curve data from /api/profiles/[profileId]/curve
    - Convert curve points to timestamp-aligned format: Each point's timestamp = session startedAt + (offsetSeconds * 1000) - curveOffsetSeconds * 1000 (accounting for partial charge offset)
    - Pass as `referenceData` prop to the PowerChart component
    - The reference curve appears as a dashed gray line behind the live blue line

    This creates a useEffect that:
    1. Listens to charge state events for this plug
    2. When state has profileId and curveOffsetSeconds, fetches the curve
    3. Aligns timestamps to the current session start time
    4. Updates referenceData state
    5. Clears referenceData when session ends (state='complete' or 'idle')
  </action>
