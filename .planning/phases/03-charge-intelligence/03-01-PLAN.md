    State machine tests:
    - Starts in IDLE state
    - Transitions IDLE -> DETECTING when power exceeds threshold for sustained period
    - Transitions DETECTING -> MATCHED when DTW match found
    - Transitions MATCHED -> CHARGING (auto after brief display period)
    - Transitions CHARGING -> COUNTDOWN when SOC reaches target - 5
    - Transitions COUNTDOWN -> STOPPING when SOC reaches target
    - Transitions any state -> IDLE on manual abort
    - Transitions IDLE -> LEARNING on learn mode start
    - Transitions LEARNING -> LEARN_COMPLETE when power drops to idle

  <action>
    **DTW (src/modules/charging/dtw.ts):**
    Implement dtwDistance and subsequenceDtw exactly as specified in RESEARCH.md code examples. dtwDistance uses a flat Float64Array cost matrix, returns normalized distance (cost / max(n, m)). subsequenceDtw slides query along reference with configurable windowStep (default 5), returns { offset, distance }.

    **SOC Estimator (src/modules/charging/soc-estimator.ts):**
    - `computeSocBoundaries(curvePoints: Array<{offsetSeconds: number, cumulativeWh: number, apower: number}>): SocBoundary[]` -- divides totalWh into 10 equal buckets, finds the curve point closest to each 10% energy boundary. Returns 10 SocBoundary objects (10%, 20%, ... 100%).
    - `estimateSoc(currentWh: number, totalWh: number, startSoc: number = 0): number` -- returns integer 0-100. For full charges (startSoc=0): `Math.round((currentWh / totalWh) * 100)`. For partial charges: `Math.round(startSoc + ((currentWh / (totalWh * (1 - startSoc / 100))) * (100 - startSoc)))`. Clamp to 0-100.

    **Charge State Machine (src/modules/charging/charge-state-machine.ts):**
    Class with:
    - `state: ChargeState` (starts 'idle')
    - `sessionId: number | null`
    - `feedReading(apower: number, timestamp: number): ChargeState` -- processes one reading, may trigger transition
    - `startLearning(sessionId: number): void` -- force transition to 'learning'
    - `setMatch(match: MatchResult, sessionId: number): void` -- force transition to 'matched'
    - `abort(): void` -- force transition to 'idle'
    - `onTransition: ((from: ChargeState, to: ChargeState, data?: unknown) => void) | null`

    Internal logic:
    - IDLE: If apower > CHARGE_THRESHOLD (5W) sustained for 30 seconds (count consecutive readings above threshold), transition to DETECTING.
    - DETECTING: Accumulate readings for up to 120 seconds. If no match callback triggers setMatch(), transition back to IDLE. The actual DTW matching is done externally by ChargeMonitor.
    - MATCHED: Auto-transition to CHARGING after 5 seconds (display time for banner).
    - CHARGING: Track cumulative Wh. When SOC >= targetSoc - 5, transition to COUNTDOWN.
    - COUNTDOWN: When SOC >= targetSoc, transition to STOPPING.
    - STOPPING: External relay controller handles actual switch-off. Transition to COMPLETE when confirmed.
    - LEARNING: Accumulate readings. When apower < IDLE_THRESHOLD (2W) sustained for 60 seconds, transition to LEARN_COMPLETE.

    Constants: CHARGE_THRESHOLD = 5, IDLE_THRESHOLD = 2, DETECTION_WINDOW_S = 120, SUSTAINED_READINGS = 6 (30s at 5s interval), LEARN_IDLE_READINGS = 12 (60s at 5s interval).

    **Relay Controller (src/modules/charging/relay-controller.ts):**
    - `switchRelayOff(mqttService, plug: {mqttTopicPrefix, ipAddress}, eventBus): Promise<boolean>`
    - Send MQTT 'off' command via mqttService.publishCommand
    - Wait 3 seconds, check if power dropped (listen for next power reading < 5W)
    - If not, retry via HTTP fetch to `http://{ipAddress}/rpc/Switch.Set?id=0&on=false` up to 3 times
    - Per CHRG-07: Export HYSTERESIS_COOLDOWN_MS = 60_000. After successful relay off, the ChargeMonitor must not allow relay on for this duration. Store `lastRelayOffAt` timestamp.
    - `canSwitchRelay(lastRelayOffAt: number): boolean` -- returns true if Date.now() - lastRelayOffAt > HYSTERESIS_COOLDOWN_MS

    Write tests for DTW, SOC estimator, and state machine. Relay controller is not unit-tested here (integration test in later plan) because it requires MQTT/HTTP mocking.
