    Build the MQTT data pipeline and custom server entry point:

    1. **src/modules/events/event-bus.ts**: Implement EventBus class extending Node.js EventEmitter (per RESEARCH.md Pattern 3):
       - Export `PowerReading` interface: `{ plugId: string, apower: number, voltage: number, current: number, output: boolean, totalEnergy: number, timestamp: number }`
       - Export `PlugOnlineEvent` interface: `{ plugId: string, online: boolean }`
       - `emitPowerReading(reading: PowerReading)`: emits on `power:<plugId>` and `power:*`
       - `emitPlugOnline(plugId: string, online: boolean)`: emits on `online:<plugId>` and `online:*`
       - `emitDiscoveredDevice(deviceId: string)`: emits on `discovery:device`

    2. **src/modules/mqtt/shelly-parser.ts**: Implement Zod-validated Shelly Gen3 payload parser (per RESEARCH.md code example):
       - `ShellyEnergySchema`: `{ total: number, by_minute?: number[], minute_ts?: number }`
       - `ShellySwitchStatusSchema`: `{ id: number, source?: string, output: boolean, apower: number, voltage: number, current: number, pf?: number, freq?: number, aenergy: ShellyEnergySchema, temperature?: { tC: number, tF: number } }`
       - Export `ShellySwitchStatus` type inferred from schema
       - Export `parseShellyStatus(payload: string): ShellySwitchStatus | null` -- JSON.parse + safeParse, return null on failure

    3. **src/modules/mqtt/discovery.ts**: Implement MQTT auto-discovery logic (per D-07):
       - Export `DISCOVERY_TOPICS`: `['shellyplugsg3-+/online', 'shellyplugsg3-+/status/switch:0']`
       - Export `parseDeviceId(topic: string): string` -- extracts first path segment (e.g., `shellyplugsg3-AABBCC/online` -> `shellyplugsg3-AABBCC`)
       - Export `isDiscoveryTopic(topic: string): boolean` -- checks if topic matches any discovery pattern
       - Export `DiscoveredDevice` type: `{ deviceId: string, firstSeen: number, lastSeen: number, online: boolean }`

    4. **src/modules/mqtt/mqtt-service.ts**: Implement MqttService class (per RESEARCH.md MQTT Service Skeleton):
       - Constructor takes `eventBus: EventBus`
       - `connect(brokerUrl: string, options?: mqtt.IClientOptions): Promise<void>` -- connects with `reconnectPeriod: 1000`, `connectTimeout: 5000`. Sets up event handlers for 'connect', 'message', 'reconnect', 'error', 'offline'.
       - On 'connect': subscribe to `DISCOVERY_TOPICS` plus topics for all registered plugs from DB
       - On 'message': parse topic to get deviceId via `parseDeviceId()`. If `/online` topic, emit `plugOnline`. If `/status/switch:0`, parse with `parseShellyStatus()`, emit `emitPowerReading`, and persist to `powerReadings` table.
       - `subscribeToPlug(plugId: string)`: subscribe to `<plugId>/status/switch:0` and `<plugId>/online`
       - `unsubscribeFromPlug(plugId: string)`: unsubscribe from plug topics
       - `testConnection(brokerUrl: string, options?): Promise<boolean>` -- create temporary client with `reconnectPeriod: 0`, resolve true on connect, false on error/timeout (5s)
       - `disconnect(): Promise<void>` -- end client gracefully
       - `isConnected(): boolean` -- return client.connected status
       - Private `persistPowerReading(reading: PowerReading)`: insert into powerReadings table using Drizzle. Import db from `@/db/client`.
       - Import `server-only` at top.

    5. **server.ts**: Custom server entry point (per RESEARCH.md Pattern 1):
       - Import `createServer` from 'http', `next` from 'next', `MqttService`, `EventBus`, `db`, `env`, and `config` schema from db/schema.
       - `main()` async function:
         a. `const app = next({ dev: process.env.NODE_ENV !== 'production' })`
         b. `await app.prepare()`
         c. Create EventBus instance
         d. Create MqttService instance with eventBus
         e. Load MQTT settings from config table (keys: `mqtt.host`, `mqtt.port`, `mqtt.username`, `mqtt.password`). If host exists, connect to `mqtt://<host>:<port||1883>`.
         f. Assign `globalThis.__eventBus = eventBus` and `globalThis.__mqttService = mqttService`
         g. Create HTTP server with `app.getRequestHandler()`
         h. Listen on `env.PORT` (default 3000)
         i. Log startup message: `"Charging Master ready on http://localhost:<port>"`
       - Handle SIGTERM/SIGINT: disconnect MQTT, close server, exit
       - Call `main().catch(console.error)`
