  <action>
Create `src/app/api/version/route.ts` implementing the locked API shape exactly. Reuse `sqlite` from `@/db/client` for the synchronous `SELECT 1` probe (better-sqlite3 is synchronous; no async/await needed).

```ts
import { CURRENT_SHA, CURRENT_SHA_SHORT, BUILD_TIME } from '@/lib/version';
import { sqlite } from '@/db/client';
import { UpdateStateStore } from '@/modules/self-update/update-state-store';

export const runtime = 'nodejs';
export const dynamic = 'force-dynamic';

type VersionResponse = {
  sha: string;
  shaShort: string;
  buildTime: string;
  rollbackSha: string | null;
  dbHealthy: boolean;
};

function probeDb(): boolean {
  try {
    // Synchronous SELECT 1 via better-sqlite3. Wrapped in try/catch so any
    // error (locked DB, missing file, permission issue) downgrades to
    // dbHealthy=false without breaking the endpoint.
    const row = sqlite.prepare('SELECT 1 as ok').get() as { ok: number } | undefined;
    return row?.ok === 1;
  } catch {
    return false;
  }
}

function readRollbackSha(): string | null {
  try {
    // The file is guaranteed to exist because server.ts main() called
    // UpdateStateStore.init() before binding the HTTP server. If it is
    // somehow missing (manual deletion mid-run), return null gracefully.
    const store = new UpdateStateStore();
    return store.read().rollbackSha;
  } catch {
    return null;
  }
}

export function GET(): Response {
  const body: VersionResponse = {
    sha: CURRENT_SHA,
    shaShort: CURRENT_SHA_SHORT,
    buildTime: BUILD_TIME,
    rollbackSha: readRollbackSha(),
    dbHealthy: probeDb(),
  };
  return Response.json(body, {
    headers: {
      // Aggressively disable caching — every request must hit the live DB probe
      // so it doubles as the post-restart health check in Phase 9 (EXEC-04 / ROLL-04).
      'Cache-Control': 'no-store, no-cache, must-revalidate',
    },
  });
}
```

Key decisions encoded above (from CONTEXT.md + PITFALLS.md):
- `export const runtime = 'nodejs'` — explicitly locked in CONTEXT.md; required because better-sqlite3 needs node:fs.
- `export const dynamic = 'force-dynamic'` — the DB probe must run on every request. Static optimization would defeat the health check.
- `GET()` is a synchronous function (not `async`). better-sqlite3 is sync, UpdateStateStore.read() is sync, there's literally nothing to await. Declaring it non-async shaves a microtask off the hot path.
- `probeDb()` wraps in try/catch and returns `false` on any error. Matches CONTEXT.md: "Wrap in try/catch → dbHealthy: false on any error."
- `readRollbackSha()` also try/catches — same rationale. If state.json is somehow corrupt or missing, the route still returns 200 (not 500) with `rollbackSha: null`. The alternative (returning 500) would brick the Phase 9 post-restart health probe which depends on HTTP 200.
- Cache-Control header prevents any intermediate proxy (or Next.js) from serving a stale response during the post-update restart window.
- No authentication, no rate-limit. LAN-only, single-user app (see CLAUDE.md constraints).

Do NOT:
- Do NOT use `NextResponse.json` — the rest of this codebase uses plain `Response.json(...)` (see src/app/api/settings/route.ts).
- Do NOT call `db.select()` on Drizzle. Use the raw `sqlite` handle directly — it's faster for a one-column probe and proves the underlying better-sqlite3 binding is loaded (which is what the Phase 9 rollback gate actually cares about: "Could not locate the bindings file" from P1/P2 in PITFALLS.md would manifest here).
- Do NOT add a `HEAD` export. Phase 9's health probe uses GET explicitly.

Current shape of `main()` (first few lines):

Changed shape:
```ts
import { UpdateStateStore } from './src/modules/self-update/update-state-store';
// ... (existing imports above this, unchanged)

async function main() {
  await app.prepare();

  // Initialize self-update state store BEFORE anything else touches it.
  // Creates .update-state/ and seeds state.json with { currentSha: CURRENT_SHA, ... }
  // on fresh installs. Idempotent: never overwrites an existing file.
  UpdateStateStore.init();

  const eventBus = new EventBus();
  const httpPollingService = new HttpPollingService(eventBus);
  // ... (rest unchanged)
```

The exact import path matters: server.ts uses relative imports (not `@/` alias) — see existing imports like `./src/modules/shelly/http-polling-service`. Match that style:

```ts
import { UpdateStateStore } from './src/modules/self-update/update-state-store';
```

Placement rationale (from CONTEXT.md "Boot integration"):
- BEFORE HttpPollingService: if state.json becomes required for a later service (Phase 8 UpdateChecker reads `lastCheckEtag` on boot), init must happen first. Ordering this now means we don't have to re-audit when Phase 8 lands.
- AFTER `await app.prepare()`: Next.js prep is cwd-independent but init() uses `process.cwd()`, so it's safe before or after. Placing after prepare() matches the pattern of "prepare Next, then bring up services".
- NOT inside the shutdown handler: init is boot-only. State is flushed atomically on every write, so there is no "drain" step to add on shutdown.

Do NOT:
- Do NOT wrap the call in try/catch. If `init()` fails (e.g. `.update-state/` is not writable), the process SHOULD crash. A running app without valid state.json would silently break every rollback attempt — fail loud, fail fast.
- Do NOT add a `try { UpdateStateStore.init() } catch (e) { console.warn(e) }` defense. See above.
- Do NOT call `init()` inside any service constructor. One process = one init call.
- Do NOT add a `.update-state/` entry anywhere else — plan 07-01 Task 1 already put it in .gitignore.

No shutdown handler changes are needed: state.json persists on disk; there is nothing in-memory to flush on SIGTERM.

Create a tiny client component at `src/app/settings/version-badge.tsx`:

```tsx
'use client';

import { useState } from 'react';
import { CURRENT_SHA, CURRENT_SHA_SHORT, BUILD_TIME } from '@/lib/version';

// Format build time as YYYY-MM-DD for compact display.
// Keep it dumb: no timezone magic, no locale formatting — BUILD_TIME is ISO UTC.
function formatBuildDate(iso: string): string {
  // '2026-04-10T12:34:56.789Z' -> '2026-04-10'
  return iso.slice(0, 10);
}

export function VersionBadge() {
  const [copied, setCopied] = useState(false);

  const buildDate = formatBuildDate(BUILD_TIME);
  const isUnknown = CURRENT_SHA_SHORT === 'unknown';

  async function handleCopy() {
    if (isUnknown) return;
    try {
      await navigator.clipboard.writeText(CURRENT_SHA);
      setCopied(true);
      setTimeout(() => setCopied(false), 1500);
    } catch {
      // navigator.clipboard requires secure context (HTTPS or localhost).
      // On a LAN IP like http://charging-master.local:3000 modern browsers
      // may refuse. Fall through silently — the full SHA is still visible
      // via the native title tooltip on hover, which is good enough.
    }
  }

  return (
    <button
      type="button"
      onClick={handleCopy}
      title={isUnknown ? 'Version unbekannt — Build ohne Git-Kontext' : `Vollständiger SHA: ${CURRENT_SHA}\nKlicken zum Kopieren`}
      className="inline-flex items-center gap-2 rounded border border-neutral-800 bg-neutral-900 px-3 py-1.5 font-mono text-xs text-neutral-300 transition-colors hover:border-neutral-700 hover:bg-neutral-800"
    >
      <span className="text-neutral-500">v</span>
      <span className="text-neutral-100">{CURRENT_SHA_SHORT}</span>
      <span className="text-neutral-500">·</span>
      <span className="text-neutral-400">{buildDate}</span>
      {copied && <span className="ml-1 text-emerald-400">Kopiert ✓</span>}
    </button>
  );
}
```

Design notes (CONTEXT.md locked: "Subtle, not a new page", "top of the existing Settings page"):
- `<button>` (not `<div>`) because it's interactive — click-to-copy. Accessible by default with keyboard + screen reader.
- `title` attribute gives the native hover tooltip with full SHA. No custom tooltip component needed.
- `font-mono text-xs` matches the compact dev-tool aesthetic; the badge should feel like a build identifier, not marketing.
- "unknown" SHA path: hides the copy affordance (nothing useful to copy) but still shows the badge so the operator sees "this build is missing git context" rather than a blank page.
- Clipboard fallback: modern browsers require secure context for `navigator.clipboard`. On `http://charging-master.local:3000` some browsers will refuse. CONTEXT.md flags this as Claude's Discretion — we prefer to fail silently and keep the hover tooltip as the fallback rather than importing a legacy `execCommand('copy')` shim for a single feature. Operator can still select-and-copy from the tooltip or DevTools.
- 1.5s confirmation visibility — long enough to notice, short enough to not linger.
- Uses Tailwind neutral palette to match existing settings components (see `src/components/settings/settings-section.tsx`).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LAN client → `/api/version` | LAN-only, single-user app, no auth (per CLAUDE.md). Anyone with network access to the LXC can call it. |
| Client bundle → `CURRENT_SHA` | Full SHA is shipped to the browser inside the client bundle (VersionBadge imports it). |
| Route handler → better-sqlite3 | `SELECT 1` probe touches the live DB; any sync lock contention affects response time. |
| Route handler → `.update-state/state.json` | Sync read on every request. File is trusted-input (written only by this process and the Phase 9 updater). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-02-01 | I (Information Disclosure) | `GET /api/version` exposing commit SHA + DB health over LAN | accept | LAN-only deployment on single-user LXC. Public repo means the SHA is not secret. DB health flag reveals only a binary "DB works" signal. No PII, no credentials, no internal topology. This endpoint's raison d'être is to be queried by external consumers (Phase 9 post-restart health gate, Phase 10 auto-reload poll). |
| T-07-02-02 | D (Denial of Service) | `GET /api/version` hit in a tight loop | mitigate | Sync `SELECT 1` is <1ms; sync state.json read is <1ms. Total <50ms under any realistic LAN load. No rate limiter — the single-user constraint means there is no realistic abuse vector on the local network. If Phase 10's auto-reload poll misbehaves, it's a local bug we'll see in dev, not a security issue. |
| T-07-02-03 | T (Tampering) | Clipboard API write from VersionBadge | mitigate | `navigator.clipboard.writeText(CURRENT_SHA)` writes a baked-in constant to the user's own clipboard. Not injection-capable: the SHA is a 40-char hex (or `"unknown"`) fixed at build time, never comes from user input or the network, and is not interpolated into any executable context. No XSS surface. |
| T-07-02-04 | S (Spoofing) | `GET /api/version` as a health probe | accept | Phase 9's post-restart probe depends on this endpoint to prove the new version is up. No authentication means anyone on the LAN could fake a 200 response (if they could somehow MITM localhost), but the probe runs over `http://localhost:3000` from the same host — no network hop to compromise. |
| T-07-02-05 | I (Information Disclosure) | `rollbackSha` field in response | accept | Exposes previous SHA after an auto-rollback. Same rationale as T-07-02-01: public repo, no secret. Phase 10 explicitly needs this to render the rollback banner. |
| T-07-02-06 | D (Denial of Service) | `UpdateStateStore.init()` failure crashes boot | accept | Intentional. An app that cannot initialize state.json cannot safely participate in self-update. Crashing on boot is louder than silently running broken — exactly the right failure mode for this class of problem. |
| T-07-02-07 | E (Elevation of Privilege) | `export const runtime = 'nodejs'` on /api/version | mitigate | Required for better-sqlite3 and fs imports. Does not elevate privilege beyond what the route handler process already has (the app runs as root in the LXC per STACK.md). No sudo/spawn calls in this route — only sync reads. |

**Conclusion:** Low risk. The endpoint is designed as a public-over-LAN health probe by explicit product requirement. No authentication needed (single-user LAN constraint, locked in CLAUDE.md). Ship.
