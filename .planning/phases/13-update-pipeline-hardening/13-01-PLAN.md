<objective>
Harden the bash updater pipeline against the two defects that bricked the 2026-05-15 v1.3.1 deploy: (a) PIPE-01 — untracked files in /opt/charging-master no longer kill preflight; they get moved to .update-state/quarantine-<ts>/ and the run continues. (b) PIPE-02 — `on_error` ALWAYS resets state.json:updateStatus to 'idle' before exiting non-zero, so a failed preflight no longer permanently strands the pipeline at 409 'already in progress'.

Wave 1 of 3. Foundation for plans 13-02..04 because the new state.json shape (`lastQuarantine?`) ships here and is referenced by plans 13-03 (admin DELETE that clears it) + 13-04 (UI banner that reads it).

Purpose: Eliminate the SSH-recovery class of failure documented in CONTEXT.md §Incident. Both defects must ship together — partial deploys (only PIPE-02 without PIPE-01) keep the same underlying preflight brittleness on the next untracked file. (Per RESEARCH Open Q7: this is why the four PIPE requirements ship as one phase.)

Output:
- run-update.sh: preflight_git rewritten to use `git status -z --porcelain` + NUL-loop, two new state.json helpers, on_error trap extended, snapshot tarball excludes quarantine dirs, retention prune.
- types.ts + update-info-view.ts: lastQuarantine added as an optional field that flows through to the UpdateInfoView the UI consumes.
- dry-run-helpers.sh: two new tests (quarantine happy-path + on_error reset) that match the existing Test 1–4 structure.
</objective>

  <files>scripts/update/run-update.sh</files>
  <action>
Replace the entire `preflight_git` function at lines 269–278 with the quarantine-aware version per RESEARCH PIPE-01 §1 + PATTERNS §1.

Key invariants the implementation MUST hold:
1. Parse via `while IFS= read -r -d '' entry; do ... done < <(git status -z --porcelain)`. Per RESEARCH Pitfall 1, NEVER use newline-delimited porcelain. The `-z` form ALSO disables C-string quoting so the entry bytes are the raw filename — no unquoting needed.
2. For each entry: `local code="${entry:0:2}"` and `local path="${entry:3}"`. The `-z` form puts a single space at byte index 2 between code and path.
3. Status code partitioning (RESEARCH Pitfall 2 + 3):
   - `??` (exactly two question marks) → untracked → quarantine candidate.
   - Anything else → fatal. This includes ` M`, `M `, `MM`, `AM`, `AD`, ` D`, `R*`, `C*`, `DD`, `AU`, `UD`, `UA`, `DU`, `AA`, `UU`, `!!`. Per Pitfall 2, rename codes (`R*` / `C*`) consume TWO NUL tokens, but since we abort fatally on the first one, we never need to consume the second.
4. Path-prefix exclusion for `??` entries: skip any path matching `.update-state/*` or `.next/*` (the bash script's own write directories — per RESEARCH Pitfall 15). Use a `case "${path}" in .update-state/*|.next/*) continue ;; esac` block. Do NOT quarantine these.
5. Accumulate two bash arrays: `untracked[]` (paths to move) and `fatal[]` (full entries with code). If `fatal[]` has any element after the read loop, call `die "Working tree has unexpected changes: ${fatal[*]}"` (mirrors the existing `die` semantics, fires on_error).
6. If `untracked[]` is empty: `log "git working tree clean"` and `return 0`. No quarantine dir created.
7. Otherwise: build the quarantine dir path. Stamp = `$(date +%Y%m%d-%H%M%S)`. Base path = `${STATE_DIR}/quarantine-${stamp}`. Per RESEARCH Pitfall 4, handle same-second re-trigger by checking `[ -e "${qdir}" ]` and appending `-tryN` (1..10). `mkdir -p "${qdir}"`.
8. For each path in `untracked[]`: `local dest="${qdir}/${f}"`, `mkdir -p "$(dirname "${dest}")"`, `mv "${INSTALL_DIR}/${f}" "${dest}" || die "quarantine move failed for ${f}"`. Per RESEARCH Pitfall 5 + 6, `mv` is atomic within the same filesystem and correctly relocates symlink nodes (does NOT follow).
9. After all moves succeed: `log "quarantined ${#untracked[@]} file(s) to ${qdir}"` (the exact wording the SSE stream parser at update-banner.tsx:38 expects to surface in journalctl per RESEARCH Pitfall 16).
10. Retention prune AFTER quarantine creation, mirroring snapshot retention at lines 289–294 (PATTERNS §4):

    local existing
    existing=$(ls -1td "${STATE_DIR}"/quarantine-* 2>/dev/null | tail -n +$((QUARANTINE_RETAIN + 1)) || true)

12. Final: `log "git working tree clean (after quarantine)"` and `return 0`.
