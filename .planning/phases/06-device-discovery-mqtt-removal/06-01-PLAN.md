Create `src/modules/shelly/discovery-scanner.ts`:

1. Export type `ScanResult`:
   ```
   { ip: string; deviceId: string; model: string; gen: number; apower: number; output: boolean }
   ```

2. Export async function `getLocalSubnet(): string` that uses `os.networkInterfaces()` (per D-02) to find the first non-internal IPv4 address and returns the /24 prefix (e.g., "192.168.3"). If no suitable interface found, throw an Error("No suitable network interface found").

3. Export async function `probeDevice(ip: string, timeoutMs: number): Promise<ScanResult | null>`:
   - Fetch `http://${ip}/rpc/Shelly.GetDeviceInfo` with AbortSignal.timeout(timeoutMs)
   - If response OK, parse JSON to get `id`, `model`, `gen`
   - Then fetch `http://${ip}/rpc/Switch.GetStatus?id=0` with same timeout
   - If response OK, parse JSON to get `apower`, `output`
   - Return ScanResult with all fields
   - On any error (timeout, network, parse), return null silently

4. Export async function `scanSubnet(options?: { concurrency?: number; timeoutMs?: number; onProgress?: (scanned: number, total: number) => void }): Promise<ScanResult[]>`:
   - Default concurrency: 20, default timeoutMs: 1500 (per D-03)
   - Call getLocalSubnet() to get prefix
   - Scan IPs 1-254 in parallel batches of `concurrency` size
   - Call onProgress after each batch completes
   - Return array of non-null ScanResult entries
   - Total scan time target: 5-15 seconds (per D-03)

Rewrite `src/app/api/devices/discover/route.ts`:
- Remove all MQTT/globalThis.__mqttService/__discoveredDevices references
- Import `scanSubnet` from `@/modules/shelly/discovery-scanner`
- GET handler: call `scanSubnet()`, return `Response.json({ devices: results })`
- Wrap in try/catch, return 500 with `{ error: "scan_failed" }` on error
- Keep `export const runtime = 'nodejs'`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> API routes | User triggers scans and device registration |
| API -> Local network devices | HTTP calls to arbitrary IPs on the subnet |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | S (Spoofing) | discovery-scanner.ts | accept | Local LAN only, single-user app, no auth by design (SETT constraint). Attacker would need LAN access. |
| T-06-02 | T (Tampering) | POST /api/devices | mitigate | Validate ipAddress is a non-empty string in the POST handler. Validate id and name are non-empty strings. |
| T-06-03 | I (Information Disclosure) | discovery-scanner.ts | accept | Subnet scan reveals which IPs have Shelly devices. Acceptable on local LAN single-user app. |
| T-06-04 | D (Denial of Service) | GET /api/devices/discover | mitigate | Scanner uses concurrency limit (20) and per-IP timeout (1.5s). Total scan bounded at ~20 seconds worst case. No concurrent scan protection needed for single-user app. |
| T-06-05 | T (Tampering) | probeDevice response | accept | Shelly device could return spoofed data. Acceptable risk -- local trusted network, no security-critical decisions based on discovery data. |
</threat_model>
