   - Export `async function sendPushover(msg: PushoverMessage): Promise<boolean>` that POSTs JSON to `https://api.pushover.net/1/messages.json` with fields `{ token: msg.apiToken, user: msg.userKey, title, message, priority }`. Return `res.ok`. Wrap in try/catch, log error and return false on failure.

2. Create `src/modules/notifications/notification-service.ts`:
   - Import EventBus, ChargeStateEvent, sendPushover, db, config table, eq from drizzle-orm.
   - Define `NOTIFICATION_STATES = new Set(['matched', 'complete', 'error', 'aborted', 'learn_complete'])` (per D-40).
   - Export `class NotificationService`:
     - `private eventBus: EventBus`
     - `private lastNotifiedState = new Map<string, string>()` -- tracks last notification state per plugId to prevent duplicates (Pitfall 1 from research).
     - `private lastNotifiedTime = new Map<string, number>()` -- cooldown: skip same type for same plug within 60 seconds (Pitfall 3).
     - `constructor(eventBus: EventBus)` stores eventBus.
     - `start()`: register handler on `eventBus.on('charge:*', handler)`.
     - `stop()`: remove listener.
     - `private handleEvent(event: ChargeStateEvent)`: Skip if `event.state` not in NOTIFICATION_STATES. Skip if `lastNotifiedState.get(event.plugId) === event.state` AND `Date.now() - lastNotifiedTime.get(event.plugId) < 60000`. Read credentials via `getCredentials()`. If null, return (D-42: skip if not configured). Build message via `buildMessage(event)`. Call `sendPushover(...)`. Update lastNotifiedState and lastNotifiedTime. On terminal states (complete, error, aborted), delete the plug from lastNotifiedState map so next session triggers fresh.
     - `private async getCredentials()`: Query config table for `pushover.userKey` and `pushover.apiToken`. Return `{ userKey, apiToken }` or null if either missing.
     - `private buildMessage(event)` returns `{ title, message, priority }`:
       - `matched`: title="Ladevorgang erkannt", message="{profileName} wurde erkannt ({confidence}% Konfidenz)", priority=0 (per D-41)
       - `complete`: title="Ziel-SOC erreicht", message="Laden abgeschlossen bei {estimatedSoc}% SOC", priority=0
       - `error`/`aborted`: title="Ladefehler", message="Ladevorgang fehlgeschlagen: {stopReason or state}", priority=1 (per D-41, high priority with alarm sound)
       - `learn_complete`: title="Lernvorgang abgeschlossen", message="Referenzkurve aufgezeichnet fuer {profileName}", priority=0

1. Create `src/modules/charging/session-recorder.ts`:
   - Import EventBus, PowerReading, ChargeStateEvent, db, sessionReadings, sessionEvents, chargeSessions, eq.
   - Export `class SessionRecorder`:
     - `private eventBus: EventBus`
     - `private activeSessionStartTimes = new Map<string, number>()` -- maps plugId to session startedAt timestamp
     - `private activeSessionIds = new Map<string, number>()` -- maps plugId to sessionId
     - `private readingCount = new Map<string, number>()` -- throttle: only persist every Nth reading
     - `constructor(eventBus: EventBus)` stores eventBus.
     - `start()`: Register two handlers:
       a) `eventBus.on('charge:*', chargeHandler)` -- on state transitions, track session start/end and insert sessionEvents rows.
       b) `eventBus.on('power:*', powerHandler)` -- during active sessions, insert sessionReadings rows.
     - `stop()`: remove both listeners.
     - Charge handler logic:
       - On `detecting`: Look up session from DB by plugId with state='detecting' to get sessionId and startedAt. Store in activeSessionIds and activeSessionStartTimes maps. Insert sessionEvents row (sessionId, state='detecting', timestamp=Date.now()). Reset readingCount for this plug.
       - On `matched`, `charging`, `countdown`, `stopping`, `learning`, `learn_complete`: If sessionId known, insert sessionEvents row.
       - On `complete`, `error`, `aborted`, `idle`: Insert sessionEvents row if sessionId known. Then clean up maps (delete plugId from all tracking maps).
     - Power handler logic:
       - If no activeSessionId for this plugId, skip.
       - Increment readingCount. Only persist every 5th reading (1 per ~5 seconds) to avoid DB bloat.
       - Calculate offsetMs = `reading.timestamp - activeSessionStartTimes.get(plugId)`.
       - Insert into sessionReadings: `{ sessionId, offsetMs, apower: reading.apower, voltage: reading.voltage, current: reading.current, timestamp: reading.timestamp }`.
